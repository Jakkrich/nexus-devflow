# 🚀 PRPs (Pure Agentic) Framework Workflows

คลังคำสั่งและเวิร์กโฟลว์มาตรฐานสำหรับการพัฒนาซอฟต์แวร์ด้วย AI (Cursor, Windsurf, Antigravity) ออกแบบตามหลักการ **Context Engineering** และ **System-Agentic Development** เพื่อให้ AI ทำงานร่วมกับคุณได้อย่างมีประสิทธิภาพสูงสุดในทุกโครงการของ Ramasri

---

## 🏗️ โครงสร้างลำดับขั้นตอน (Workflow Phases)

เราจัดกลุ่มคำสั่งตามวงจรการพัฒนาซอฟต์แวร์ (Software Development Lifecycle) เพื่อให้ง่ายต่อการเรียกใช้งาน:

### **Phase 00: การเตรียมการและตั้งต้น (Initialization & Setup)**
*   **`/00-Init`**: เริ่มต้นโปรเจกต์ใหม่หรือซิงค์บริบทของ Workspace ปัจจุบัน
*   **`/01-App-Builder`**: ตัวช่วยสร้างโครงสร้างโปรเจกต์ใหม่ (Scaffolding) จากศูนย์
*   **`/02-Status`**: ตรวจสอบสถานะความคืบหน้าของงานและสถานะของ Agent ทั้งหมด

### **Phase 10: การวิเคราะห์และออกแบบ (Discovery & Design)**
*   **`/10-Brainstorm`**: ระดมสมองและเปรียบเทียบทางเลือกในการแก้ปัญหาเชิงสถาปัตยกรรม
*   **`/11-Research`**: เจาะลึกโค้ดเบสเดิม หรือวิจัยเทคโนโลยีใหม่พร้อมหลักฐานอ้างอิง
*   **`/12-PRD`**: ร่างเอกสารความต้องการ (PRD) และเกณฑ์การยอมรับ (Acceptance Criteria)
*   **`/13-UI-UX`**: ออกแบบหน้าตาและประสบการณ์ผู้ใช้ด้วยระบบ Design Intelligence
*   **`/14-Orchestrate`**: วางแผนงานที่ซับซ้อนโดยใช้ทีม Agent หลายตัวทำงานร่วมกัน

### **Phase 20: การวิเคราะห์ปัญหา (Problem Solving)**
*   **`/20-Debug`**: วิเคราะห์หาสาเหตุที่แท้จริง (Root Cause Analysis) ด้วยเทคนิค 5-Whys

### **Phase 30: วงจรการพัฒนาหลัก (Core Execution Cycle)**
*   **`/30-Task`**: เปิดงานใหม่ ย่อยงาน และสร้างพื้นที่สำหรับติดตามผล (Workspace)
*   **`/31-Plan`**: วางแผนการเขียนโค้ดอย่างละเอียดแบบทีละขั้นตอนก่อนเริ่มลงมือ
*   **`/32-Code`**: ลงมือเขียนโค้ดจริงแบบทีละส่วน (Incremental) พร้อมระบบตรวจสอบอัตโนมัติ
*   **`/33-Verify`**: ตรวจสอบคุณภาพโค้ดใน 5 มิติหลัก (Senior QA Gate)
*   **`/34-Human`**: ส่งงานให้มนุษย์รีวิว อนุมัติ หรือส่งกลับไปแก้ไขตามความต้องการ

### **Phase 40: คุณภาพและการเพิ่มประสิทธิภาพ (Quality & Optimization)**
*   **`/40-Test`**: สร้างและรันชุดทดสอบ (Unit/Integration Test) แบบอัตโนมัติ
*   **`/41-Simplify`**: ปรับปรุงโค้ดให้สะอาด (Clean Code) และดูแลรักษาง่ายขึ้น
*   **`/42-Preview`**: จัดการ Preview Server เพื่อดูผลลัพธ์การทำงานจริงในเบราว์เซอร์

### **Phase 50: การส่งมอบและปล่อยตัว (Ship & Release)**
*   **`/50-Commit`**: จัดการ Stage ไฟล์และเขียน Commit Message ตามมาตรฐาน Git
*   **`/51-PR`**: สร้าง Pull Request พร้อมสรุปสิ่งที่เปลี่ยนแปลง
*   **`/52-Deploy`**: ติดตั้งแอปพลิเคชันขึ้นเซิร์ฟเวอร์ (Production/Staging)
*   **`/53-Changelog`**: บันทึกประวัติการเปลี่ยนแปลงตามเวอร์ชันของซอฟต์แวร์

## 🛤️ คู่มือเส้นทางการทำงาน (Suggested SOP Paths)

เพื่อให้ได้ผลลัพธ์ที่เป็นมาตรฐานสากล แนะนำให้เดินตามเส้นทางเหล่านี้:

### **1. 🐞 การแก้บั๊ก (Bug Fix Flow)**
เน้นการหาสาเหตุที่แท้จริงก่อนลงมือแก้ เพื่อความยั่งยืนของระบบ
> **`/20-Debug`** ➡️ **`/30-Task`** ➡️ **`/31-Plan`** ➡️ **`/32-Code`** ➡️ **`/33-Verify`** ➡️ **`/34-Human`** ➡️ **`/53-Changelog`** ➡️ **`/50-Commit`**

### **2. 🚀 การเพิ่มฟีเจอร์ใหม่ (New Feature Flow)**
เน้นการวางแผนและการออกแบบที่รอบคอบก่อนเขียนโค้ด
> **`/10-Brainstorm`** ➡️ **`/12-PRD`** ➡️ **`/30-Task`** ➡️ **`/31-Plan`** ➡️ **`/32-Code`** ➡️ **`/33-Verify`** ➡️ **`/34-Human`** ➡️ **`/53-Changelog`**➡️ **`/50-Commit`**

### **3. ⚡ การปรับปรุงคุณภาพโค้ด (Refactor Flow)**
เน้นการทำให้โค้ดสะอาดขึ้นโดยที่ฟังก์ชันการทำงานยังคงเดิม
> **`/11-Research`** ➡️ **`/41-Simplify`** ➡️ **`/33-Verify`** ➡️ **`/34-Human`** ➡️ **`/53-Changelog`**➡️ **`/50-Commit`**

## 💡 เคล็ดลับเพื่อประสิทธิภาพสูงสุด
1. **รันตามลำดับ Phase**: ลำดับตัวเลขถูกออกแบบมาเพื่อให้ AI รักษา Context ของงานได้ดีที่สุด (อย่าข้ามขั้นตอนการวางแผน)
2. **ระบุ ID ทุกครั้ง**: ในคำสั่ง Phase 30 เป็นต้นไป ให้ระบุ ID ของงานเสมอ (เช่น `/32-Code 007`) เพื่อให้ AI โฟกัสถูกงาน
3. **ตรวจสอบ Report**: หลังจากจบแต่ละ Phase AI จะสร้างรายงานในโฟลเดอร์ `.workspaces/` โปรดตรวจสอบความถูกต้องเสมอก่อนไปต่อ
4. **สื่อสารกับ Coach**: หากไม่แน่ใจว่าต้องทำอะไรต่อ สามารถเรียกใช้ `/99-Coach` เพื่อขอคำแนะนำได้ตลอดเวลา
