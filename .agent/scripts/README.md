# 📜 Antigravity Kit Scripts (สคริปต์เสริมสำหรับ Antigravity)

โฟลเดอร์นี้ประกอบด้วยสคริปต์เสริมที่ออกแบบมาเพื่อเพิ่มประสิทธิภาพในการทำงาน และช่วยตรวจสอบความถูกต้องของโค้ดโดยอัตโนมัติสำหรับทั้ง AI Agent และนักพัฒนา

## 📁 สคริปต์หลักและส่วนประกอบ

### 🧭 PRP Dashboard Contract (Node / npm)
- **`prp.mjs`**: CLI หลักสำหรับสร้าง task, update state, append event log, validate และ repair JSON contract
  - *การใช้งาน local ใน repo นี้*: `npm.cmd run agent -- validate 001`
  - *ทดสอบ CLI*: `npm run test:prp`

### 🔄 การจัดการ Preview และ Session
- **`auto_preview.py`**: ใช้จัดการ Local Development Server (เปิด/ปิด/ดูสถานะ)
  - *การใช้งาน*: `python .agent/scripts/auto_preview.py start [port]`
- **`session_manager.py`**: วิเคราะห์สถานะโปรเจกต์, ตรวจสอบ Tech Stack และสรุปข้อมูล Session ปัจจุบัน
  - *การใช้งาน*: `python .agent/scripts/session_manager.py status`

### 📋 การตรวจสอบความถูกต้อง (Validation & Verification)
- **`checklist.py`**: รันสคริปต์ตรวจสอบพื้นฐานตามลำดับความสำคัญ (Security, Lint, Tests, SEO, UX) เหมาะสำหรับการตรวจสอบความเรียบร้อยระหว่างการพัฒนา
  - *การใช้งาน*: `python .agent/scripts/checklist.py .`
- **`verify_all.py`**: ชุดเครื่องมือตรวจสอบแบบเต็มรูปแบบ รวมถึงการทดสอบประสิทธิภาพ (Lighthouse) และ E2E (Playwright) เหมาะสำหรับรันก่อนการ Deployment หรือ Release สำคัญ
  - *การใช้งาน*: `python .agent/scripts/verify_all.py . --url <URL>`

## 🤖 สำหรับ AI Agents (Antigravity)

Antigravity ควรเรียกใช้เครื่องมือเหล่านี้เพื่อรักษาคุณภาพของโค้ดและรายงานผลลัพธ์ที่เป็นระบบให้กับผู้ใช้:

1. **เริ่มต้นโปรเจกต์ (Onboarding)**: ใช้ `session_manager.py status` เพื่อทำความเข้าใจ Tech Stack และโครงสร้างโปรเจกต์อย่างรวดเร็ว
2. **ตรวจสอบระหว่างพัฒนา (Incremental Validation)**: หลังจากเพิ่มฟีเจอร์ใหม่ ควรใช้ `checklist.py .` เพื่อให้มั่นใจว่าไม่มีข้อผิดพลาดด้านความปลอดภัยหรือคุณภาพโค้ด
3. **เตรียมตัว Deployment (Deployment Readiness)**: ก่อนที่จะส่งงานหรือใช้คำสั่ง "SHIP" ควรใช้ `verify_all.py` เพื่อตรวจสอบแอปพลิเคชันอย่างละเอียด
4. **ควบคุม Dev Server**: ใช้ `auto_preview.py` เพื่อจัดการ Server ในพื้นหลัง แทนการใช้คำสั่ง `npm run dev` แบบปกติ เพื่อให้ติดตามสถานะได้ง่ายขึ้น
5. **คุม Dashboard JSON**: ใช้ `npm.cmd run agent -- validate {ID}` ทุกครั้งก่อนจบ phase และใช้ `npm.cmd run agent -- repair {ID}` เมื่อ key/file หาย

สคริปต์เหล่านี้ถูกออกแบบมาให้ AI เรียกใช้ผ่าน `run_command` เป็นหลัก แต่คุณสามารถเรียกใช้งานเองผ่าน Terminal ได้เช่นกัน:

```powershell

# ตัวอย่างการ repair JSON contract
npm.cmd run agent -- repair 001
```
