# 📜 PRPs Specialist Scripts

This folder contains supplementary scripts and utilities that help AI Agents interact with external systems and manage the PRPs-Framework lifecycle.

---

## 📁 Key Scripts & Components

### 🔄 Framework Update Utility
- **`update-prp.ps1` (PowerShell)**: Automatically updates the PRPs-Framework engines and templates for Windows users.
- **`update-prp.sh` (Bash)**: Update script for Linux, macOS, and WSL users.
- **Usage**: Typically called during initial setup or when a framework update is detected.

### 🔴 Redmine Integration (`redmine/`)
A set of tools for connecting to the **Redmine** project management system.
- `get_issue.py`: Fetch issue details.
- `search_issues.py`: Search for issues based on specific conditions.
- `update_issue_note.py`: Log progress or add notes to a Redmine ticket.
- `sync_description.py`: Synchronize `spec.md` content with the Redmine issue description.

## 🔄 Multi-IDE Support
Scripts in this folder are designed to be IDE-agnostic. If you are switching between Cursor, Antigravity, or Windsurf, use the central switcher script at the project root:

- **`active-ide.py`**: Manages the folder renaming and internal path updates for the entire framework, including references to this scripts folder.

## 🛠 Usage Guide (For Developers)

These scripts are primarily designed for Agents to invoke via `run_command`, but they can be used manually:

1. **Setup Environment**:
   ```powershell
   # Example for Redmine scripts
   cd scripts/redmine # Path is relative to active IDE root
   pip install -r requirements.txt
   ```

2. **Configuration**:
   Ensure you have configured the necessary `.env` files (e.g., Redmine API keys) as described in sub-folder documentation.
